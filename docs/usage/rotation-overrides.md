# Rotation Overrides

Rotation overrides temporarily replace the calculated on-call user.

Use overrides for:

- vacation;
- planned maintenance;
- shift swaps;
- one-off coverage changes;
- temporary absence.

## How overrides work

An override belongs to a rotation and has a time window. When the calendar or alert assignment is calculated inside that time window, the override user is used instead of the normal rotation member.

## Best practices

- Keep override windows as short and precise as possible.
- Add a clear reason.
- Verify the calendar after creating an override.
- Remove or adjust expired or incorrect overrides.
